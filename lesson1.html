SQL Lesson 1: SELECT queries

To retrieve data from an SQL database, we use SELECT statements, also called queries. A query specifies what data we need, where to find it, and optionally, how to modify it before returning the results. In SQL, a table represents an entity (e.g., "Students"), where each row is a specific instance (e.g., John, Sarah). The columns store attributes shared by all instances (e.g., age, grade, subject). The simplest query selects specific columns (attributes) from a table while retrieving all rows (instances). Let's explore how this works in the exercises ahead.

FROM mytable;

A query returns a table with the selected columns. To get all columns, use * instead of listing them. For example, SELECT * FROM employees; fetches everything from the "employees" table. This makes queries easier but may include extra data, which can affect performance.

Select query for all columns

SELECT *
FROM mytable;

This query, in particular, is really useful because it's a simple way to inspect a table by dumping all the data at once.

Common SQL Operators for Text Data

When writing WHERE clauses for text columns, SQL provides useful operators for case-insensitive comparisons and pattern matching.

= (Equals)

Exact match (case-sensitive in some databases).

SELECT * FROM users WHERE name = 'John';

ILIKE (Case-insensitive match)

Available in PostgreSQL.

SELECT * FROM users WHERE name ILIKE 'john';

LIKE (Pattern matching with wildcards)

% matches any number of characters.

SELECT * FROM users WHERE name LIKE 'Jo%'; -- Matches 'John', 'Jonathan'

_ matches a single character.

SELECT * FROM users WHERE name LIKE 'J_hn'; -- Matches 'John', 'Jahn'

IN (Multiple value match)

SELECT * FROM users WHERE name IN ('John', 'Alice', 'Bob');

NOT LIKE (Excludes pattern matches)

SELECT * FROM users WHERE name NOT LIKE 'Jo%';
