SQL Lesson 2: Queries with constraints

Now that we know how to select specific columns from a table, dealing with large datasets can be challenging. If a table has millions of rows, reading all of them would be inefficient. To retrieve only the needed data, we use the WHERE clause. It filters rows by checking column values and returns only those that meet the specified conditions.

Select query with "Where" constraints
SELECT column, another_column, …
FROM mytable
WHERE condition
  AND/OR another_condition
  AND/OR …;

SQL Logical Operators

More complex conditions can be created by combining multiple AND or OR logical operators. For example:

SELECT * FROM vehicles WHERE num_wheels >= 4 AND doors <= 2;

Useful SQL Operators

  • = → Equals (e.g., age = 25)
  • != or <> → Not equal (e.g., price != 100)
  • > → Greater than (e.g., salary > 50000)
  • < → Less than (e.g., stock < 50)
  • >= → Greater than or equal to (e.g., rating >= 4.5)
  • <= → Less than or equal to (e.g., discount <= 10)
  • BETWEEN → Range selection (e.g., age BETWEEN 18 AND 30)
  • IN → Matches any value in a list (e.g., category IN ('Electronics', 'Clothing'))

These operators help refine query results efficiently.

Did you know?

SQL does not require keywords to be written in all capital letters, but it is a common convention. Using uppercase for SQL keywords helps differentiate them from column and table names, making queries easier to read and understand.

Common SQL Operators for Text Data

When writing WHERE clauses for text columns, SQL provides useful operators for case-insensitive comparisons and pattern matching.

= (Equals)

Exact match (case-sensitive in some databases).

SELECT * FROM users WHERE name = 'John';

ILIKE (Case-insensitive match)

Available in PostgreSQL.

SELECT * FROM users WHERE name ILIKE 'john';

LIKE (Pattern matching with wildcards)

% matches any number of characters.

SELECT * FROM users WHERE name LIKE 'Jo%'; -- Matches 'John', 'Jonathan'

_ matches a single character.

SELECT * FROM users WHERE name LIKE 'J_hn'; -- Matches 'John', 'Jahn'

IN (Multiple value match)

SELECT * FROM users WHERE name IN ('John', 'Alice', 'Bob');

NOT LIKE (Excludes pattern matches)

SELECT * FROM users WHERE name NOT LIKE 'Jo%';
